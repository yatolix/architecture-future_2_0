# Роадмап трансформации технологического ландшафта Будущее 2.0

Файл с roadmap - `Roadmap.drawio`

## Roadmap создан отталкиваясь от целей бизнеса:

**Промежуточное состояние (через пару месяцев)**

* Архитектурное решение по трансформации сформировано и согласовано.
* Границы доменов определены, а ключевые связи между ними описаны.
* В доменах запланированы конкретные проекты по развитию «витрины данных».

**Финальное состояние (через год)**

* Реализован портал самообслуживания.
* Бизнес-пользователи в доменах могут работать с данными в рамках новой архитектуры.
* На этом этапе допускается временное сохранение легаси-систем в отдельных доменах.

## Этап 1 (0-2 месяца) - Закладка Фундамента

На Этапе 1 закладываем базу для всей миграции. Без согласованной архитектуры и облачных ресурсов невозможно масштабирование подхода и сервисов. Определение доменов даёт возможность командам начать независимую работу. Продумывание и прототипирование MDM на ранней стадии предотвратит создание дублей в будущем.


| Шаги                                                                                                                                                                                         | Ожидаемые результаты                                                                | Ответственные                                      | Необходимые ресурсы                                      |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| Формирование целевой архитектуры (домены, потоки, технологии)                                                                                | Согласованная архитектура (C4, DFD), технический радар         | Архитектор, СТО, Владельцы доменов | Время ключевых экспертов                             |
| Выбор облачного провайдера и настройка базовой инфраструктуры (VPC, IAM, сети)                                                        | Подписанный контракт с облаком, готовая sandbox-среда           | Инфра + isec                                               | Бюджет на пилотное использование облака |
| Развёртывание базовых компонентов платформы данных: S3, Kafka (managed), Airflow (managed) в облаке                                         | Работоспособные сервисы для разработки                              | Data Platform                                                   | Инженеры по данным, DevOps                                 |
| Запуск MDM-инициативы: определение владельцев мастер-данных, выбор решения (например, Profisee или самописное) | Прототип MDM, первые справочники (пациенты, врачи, услуги) | MDM Team                                                        | Аналитики, разработчики                               |
| Планирование миграции данных из legacy DWH (инвентаризация, приоритеты)                                                                      | План миграции по доменам                                                          | Data Platform, владельцы доменов                | Инженеры данных, аналитики                          |

## Этап 2 (2–5 месяцев) - Базовые интеграции и первые Data Products

Поэтапное подключение доменов позволит проверить подход и архитектуру на реальном примере. Бизнес-пользователи получают возможность самостоятельно строить элементарные отчёты. MDM устраняет расхождения в данных. Всё это позволяет быстрее получать инсайты из бизнес данных.


| Шаги                                                                                                                                                                                         | Ожидаемые результаты                                                                                                                                   | Ответственные                                          | Ресурсы                                        |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------- | ------------------------------------------------------- |
| Миграция выбранного домена (например, финтех) на событийную модель: подключение к Kafka, публикация событий | Первый поток данных в Kafka из финтех-домена                                                                                              | Владельца домена Финтех, Data Platform         | Разработчики финтеха               |
| Разработка ETL-пайплайнов в Airflow для загрузки данных из Kafka в Lakehouse (Iceberg)                                                                  | Автоматическая загрузка витрин финтех-данных в Lakehouse                                                                         | Data Platform                                                       | Инженеры данных                         |
| Настройка Dremio и DataHub для доступа к данным финтех-домена                                                                                             | Бизнес-пользователи финтеха могут выполнять ad-hoc запросы через Dremio (тестовый режим)                  | Data Platform, представители домена финтех | Возможно лицензии Dremio              |
| Пилотный запуск MDM для справочников "Пациенты" и "Врачи" с интеграцией в медицинский домен                              | Единый источник данных о пациентах и врачах, устранение дублей                                                       | MDM Team, представители домена                   | Интеграционные разработчики |
| Разработка минимальной версии портала самообслуживания (React) с подключением к Dremio                                           | Прототип портала для ограниченной группы пользователей (только чтение существующих витрин) | Frontend Team                                                       | UI/UX дизайнеры                              |

## Этап 3 (5–8 месяцев) - Расширение на медицинский домен и ИИ

Портал становится полноценным инструментом для бизнес пользователей. К аналитическим данным добалвяется Медицина что сильно расширяет возможности изучения данных и принятия скорых бизнес решений. ИИ сервисы добавляют свои результаты в DWH и бизнес получает возможность применять их.


| Шаги                                                                                                                                                                                                     | Ожидаемые результаты                                                                                                                         | Ответственные                                                      | Ресурсы                                                      |
| :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| Разработка Data Products медицинского домена: обезличенные витрины (загрузка врачей, статистика приёмов) без PHI            | Медицинские агрегаты в Lakehouse                                                                                                            | Представители медицинского домена, Data Platform | Медицинские аналитики, разработчики |
| Интеграция ИИ-сервисов: публикация обезличенных результатов анализа в Kafka, запись в Lakehouse                                          | Витрины с результатами работы ИИ (например, точность, количество обработанных снимков) | AI Team                                                                         | Инженеры ML, DevOps                                         |
| Расширение портала: добавление витрин медицины и ИИ, улучшение UI, внедрение DataHub для поиска данных                          | Пользователи из медицины и финтеха могут работать в портале                                                  | Frontend                                                                        | UI/UX, тестировщики                                     |
| Начало миграции данных из legacy DWH в Lakehouse по второстепенным направлениям (финансовая отчётность, инвентаризация) | Снижение нагрузки на старый DWH                                                                                                         | Data Platform                                                                   | Инженеры данных                                       |

## Этап 4 (8–12 месяцев) - Масштабирование и подготовка к новым бизнесам

Завершающий этап готовит бизнес к масштабированию чтобы можно было по устаканившимся стандартам подключать новые направления (фарма, производство оборудования). Завершается отключение неиспользуемых ресурсов для экономии средств.


| Шаги                                                                                                                                                                         | Ожидаемые результаты                                                                               | Ответственные             | Ресурсы                                                                   |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- | ---------------------------------------- | ---------------------------------------------------------------------------------- |
| Полный перевод отчётности Power BI на источник Dremio (отключение от legacy DWH)                                                    | Вся регулярная отчётность строится на данных из Lakehouse                    | BI Team                                | Аналитики BI, Инженеры DWH                                      |
| Отключение неиспользуемых частей legacy DWH и шины Camel (после верификации всех потоков)                          | Снижение затрат на поддержку устаревшего ПО                                     | Инфраструктура           | -                                                                                |
| Подготовка документации и шаблонов для подключения новых доменов (фармацевтика, производство)   | Стандарты интеграции (события, Data Products)                                               | Архитектор                   | Время архитекторов и помощь специалистов DWH |
| Обучение бизнес-пользователей работе с порталом и DataHub                                                                             | Повышение цифровой грамотности, самостоятельное использование | Data Platform                          | Обучающие материалы                                            |
| Финальное тестирование производительности и безопасности (нагрузочное тестирование, пентесты) | Подтверждение готовности системы к росту                                          | Security, Инфраструктура | Инструменты тестирования                                  |
